算术操作符

+加 -减 *乘 /除 %取余

加减乘并没有太深刻的讲解点，完全符合数理逻辑上的运算。只是由于数据类型的限制，有一些细微的差别。

主要讲解一下除和取余的操作。

除：

两个操作数都为整数时，执行整数除法。若是至少有一个浮点数则执行浮点数除法。

例如

10/3的结果是3，10.0/3的结果是3.3333····

取余：
取余操作的两个操作数都必须为整数，返回整除之后的余数。

移位操作符

例如一个十进制数123，等于1 10^2^+1 10^1^+1 10^0^，这里的10代表权重，进制是多少，权重就是多少。且权重的次方从0到2（总是在末位从0开始，到最高位）。
二进制的111则可以表示为1 2^2^+1 2^1^+1 2^0^,相加得到7，7则是二进制的123转化到十进制后的数字。

在介绍移位操作符前先简单介绍下原码，补码，反码。这三个码都在导论和原理中有所涉及。想深入了解请自行百度。
对于正数而言，原码等于补码等于反码。原码由数的二进制来表示
例如1，其原码为0000…………001（共32位，因为int为4字节，共32位）

第一位0代表的是符号，0为正，1为负。

其补码与反码也同于原码。

而对于负数而言，反码等于原码除符号位外其他位按位取反。补码则等于反码+1即可。

例如-3,其二进制表示为11，代表符号位的1置于首位。

原码 1 0 0 0 0 0 1 1（中间省略了很多0）

反码 1 1 1 1 1 1 0 0

补码 1 1 1 1 1 1 0 1

以上就是关于原码等的基础知识，是必须掌握的内容哦。
接下来介绍移位操作符。

左移 <<

右移 >>

移位操作符只有对整数有作用。

左移规则：左边抛弃，右边补0

例如num=10，使得num<<1

注意：内存中存储的是补码，则是补码在参与移位。

10的补码: 0 0 0 0 0 1 0 1 0

左移一位: 0 0 0 0 1 0 1 0 0

右移规则：

逻辑移位：左边填0，右边丢弃
算术移位：左边用该值符号位填充，右边丢弃

例如num=-1 使得num>>1

-1
原码 1 0 0 0 0 0 1

补码 1 1 1 1 1 1 1

逻辑移位：0 1 1 1 1 1 1 1

算术移位：1 1 1 1 1 1 1 1

一般而言我们使用的是算术移位。

不过对于移位运算符而言，不要移动负数位，这个是标准未定义的,可能会出现难以预料的情况。

位操作符

位操作符也是对整数进行操作。

按位与 &
按位或 |
按位异或 ^

先来看按位与。

按位与：将两数的原码每一位进行比较，只有相同权重位置上都是1才能是1，不然则为0。

1的补码 0 0 0 0 1

2的补码 0 0 0 1 0

按位与后得到00000，也就是0了。

按位或：两位有一个是1即可得到1了。

按位或后 00011，也就得到了3。

按位异或：两位相同为0，不同则为1。

那么得到的就是 00011，答案仍然是3

同时需要知道的是，按位异或满足交换律。例如5^3^5 =5^5^3。

来思考一道题目，如何在不创建第三个变量的前提下交换两个值？

法一：a=a+b,b=a-b,a=a-b

这个方法很容易想到，先让a的值等于两数总和，再让两数和减b的值就得到了原来a的值，再赋给b，a也是同理得到。

法二：

a^a=0

a^0=a

那么可以得到a=a^b b=a^b a=a^b

将a=a^b代入b=a^b中的a，得到b=a^b^b，就使得b=a。再将b=a^b代入到a=a^b中的b里去。a就等于了原本的b。

赋值运算符

整体的运用很简单。

也就是复合运算符需要提一嘴。

逻辑操作符

&& 逻辑与
|| 逻辑或

主要是要区别逻辑或与按位或，逻辑与与按位与

例如

1&& 2 ——>1
1 & 2——->0

逻辑与表达的是a&&b&&c·····，当a，b，c····都不等于0，才会使整个式子表达为1，否则则为0。
而按位与上文已经有过介绍了。在此不多赘述。

1|| 2—->1
1 |2 —->3

逻辑或则是a，b，c····中有一个为真（不等于0），即可让整个表达式为1。

需要注意的是，c语言规定，虽然非0都为真，但是逻辑表达式若为真则规定返回1。

隐式类型转换

表达式中的字符和短整型操作数在使用前需要被转换为普通整型。这被称为整型提升。

因为表达式的整型运算通常是在cpu的相应运算器件内执行，所以ALU的操作数字节长度一般为int的字节长度。所以两个char相加需要先转换为整型的标准长度。

例

char a=3

补码：00000011 （char字节只有8位，int为32位）

提升为 0000········011（共32位）

插入 b=127

补码：01111111

提升为 0000······01111111（共32位）

相加：0000·····10000010

进行截断，只要最后8位放入char型，得到了10000010。

但这是补码，所以转换为原码，11111110 ，得到了-126。

运行结果也就是-126。

这就是整型提升带来的结果。并非是我们所设想的130。

对于正数来说：
其整型提升时，高位补充符号位。即补0。
对于负数来说：
高位补充符号位。即补1。

同时需要注意的是，只要参与了表达式的运算，就会发生整型提升。
来看代码示例。

char c = 1;
printf(“%d\n”, sizeof(c));
printf(“%d\n”, sizeof(+c));
printf(“%d\n”, sizeof(-c));

因为+c，-c成了表达式，所以被提升成了4字节的int型。这是需要注意的地方。

算术转换

如果某个操作符各个操作符属于不同类型，那么除非其中一个操作数的转换为另一个操作数的类型，否则就无法进行下一步的运算了。

下表是运算的优先转换表，下面的在和上面的进行计算时需要转换为上面的。

long double
double
float
unsigned long int
long int
unsigned int
int

但是需要注意的是，转换必须要合理，否则会出现类似精度丢失的问题哦。

例如将float强制换为int，小数位就会丢失。

优先级的一些注意事项

这些操作符通常都会在表达式里成堆的出现，所以难免会有优先级的问题。

还有一些需要自己在遇到时具体分析具体讨论咯。

但是这仍然防止不了问题表达式的出现。

何谓问题表达式呢？如果表达式无法通过操作符的属性确定唯一的计算路径，那么我们就可以认为这个表达式是存在问题的了。

例如

c + —c
优先级只能确定—在+的前面，却无法确定+的左操作数的获取在右操作数前还是后进行求值，其结果存在歧义。

类似的还有

ab+cd+ef
只能确定优先于+，但如果abcdef这些存在着互相影响的因素，那么答案也会有所差别。

函数的多次调用，++ —前置后置的复杂排放等，都会导致问题表达式的出现，所以我们尽量让自己的代码不要出现这些问题。尽可能的精简易读，而非无意义的炫技。
